# Changelog Draft — March 10 to May 5, 2026

Audit of all PRs merged to `main` between **March 10, 2026** and **May 5, 2026**, organized for the public changelog. PR numbers are referenced inline for traceability.

---

## Major Updates

### Team Plans, Bulk Invites, and Seat-Based Billing
camelAI now supports a true Team plan with Stripe-synced seats. Admins can invite multiple teammates at once via a new chip-style email input, the dialog shows live billing impact (covered vs. paid seats) before submission, and seat count syncs upward automatically as members are added or removed. Plan changes and cancellations are handled through the Stripe customer portal, and a new "Copy invite link" action plus a confirmation modal for member removal round out the team management flow. (#538, #553, #554, #557, #558, #561, #563, #559, #519)

### Onboarding Paywall with Free Credits and BYOK
The signup flow now ends in a redesigned paywall with three plan tiers, a limited free-credit grant, and a "Bring Your Own Key" path through OpenRouter for users who'd rather use their own provider. Trial checkout works for every paywall plan, and the welcome screen / book-a-demo flow has been polished. (#535, #539, #540, #487)

### Subscription Credits and Enterprise Billing
camelAI hosted models are now metered against monthly included credits (Starter $10, Pro $30, Team $10/seat) with one-time trial grants and a separate purchased-credit balance. Stripe webhooks reconcile credits idempotently on `invoice.payment_succeeded`, admins can manually grant credits, and an `enterprise` billing status bypasses Stripe entirely for negotiated contracts. (#530, #541)

### Codex / GPT Models in Web Chat
Codex/GPT models are now selectable in the web chat picker behind an experiment flag, with full MCP and prompt parity with the Claude harness. Codex threads run through a warm per-thread `codex app-server`, route through an OpenAI-compatible gateway path, and load history directly from the sandbox host. Default proxy model remains Claude Sonnet. (#480, #475, #510, #511, #498, #504, #513)

### External MCP Server with OAuth 2.1
A public-facing MCP server is now available for third-party integrations, authenticated via OAuth 2.1 with `.well-known` discovery, dynamic-but-hardcoded client IDs, and `resource_metadata` in WWW-Authenticate headers. The external API was migrated off Cap'n Web RPC to a plain REST surface and moved into React Router routes. (#441, #536)

### Custom Domains: Wildcard, Self-Service, and Agent-Managed
Custom domains were rewritten to use org-scoped wildcard Cloudflare hostnames with DCV delegation, so adding a new domain no longer requires per-app provisioning. The agent can now read setup instructions, retry validation, and refresh DCV records via new MCP tools, and the settings UI was redesigned with clearer status, inline errors, and an HTTP validation fallback. SaaS custom hostnames now route through the dispatcher worker. (#420, #421, #423, #434, #454, #503, #508, #518, #520, #522, #523, #525, #526, #527, #528, #544)

### Admin API & Dashboard Metrics
A large suite of bearer-authenticated admin endpoints landed: daily-spend, top-orgs, retention curves, spam summary, per-org spend, thread JSONL export, user signup IP, BYOK org visibility, and a credit-adjustment endpoint. The dashboard pulls from a new `AdminIndexDO` membership read-model, and admin actions (ban, purge, refresh custom domain) now bypass the org-ban check so investigations aren't blocked. (#436, #443, #458, #461, #476, #488, #509, #514, #520, #531, #541, #555)

### Onboarding to In-Chat Connections
A new `@`-mention menu in the chat composer lets users pick from existing connections inline without leaving the input. Connection cards were also simplified to a single-row icon-only layout, and the "New chat" button was streamlined with hover-reveal removed in favor of always-visible icons. (#547, plus connection-card iteration)

### Canvas Spreadsheet & Markdown Outputs
Notebook outputs now render markdown content (with semantic whitespace preserved) and CSV/spreadsheet outputs render as a canvas preview rather than raw text. Both formats are included in report/PDF exports. (#545, #569)

### Settings Visual Refresh
The Billing, Usage, AI Provider, BYOK modal, Custom Domains, and Paywall surfaces all received styling passes for a denser, more consistent look — including a redesigned credit-status card with progress bar, low-credit and exhausted states, and inline `View usage` / `Top up` CTAs. (#539, #542, #544, #548, #549)
